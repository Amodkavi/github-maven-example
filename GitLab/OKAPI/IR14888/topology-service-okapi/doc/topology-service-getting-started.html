Authentication

To get an OAuth 2.0 Authentication token, pass through your Consumer Key and Consumer Secret that you received when you registered for the token-service API key. The grant_type should be left as ‘client_credentials’ and the scope as 'TOPOLOGYREAD'. The token will expire in one hour.

Get topology

The service topology can be retrieved via the /topologies resource by specifying the serviceId. When the provided service can be located, the associated topology will be returned as a HTTP 200 response, otherwise one of the below status codes will be returned to indicate the result.

The topology data is returned as per the type defined in the swagger.

HTTP Status Codes

Status Code Description
200 OK Successful request
400 Bad Request Invalid or missing request parameters
401 Unauthorized Invalid or no credentials passed in the request
403 Forbidden Authorization credentials passed and accepted but account does not have permission
404 Not Found The requested URI does not exist
500 Internal Server An internal error occurred when processing the request
501 Method Not Implemented The HTTP method being used has not yet been implemented for the requested resource
503 Service Unavailable The service requested is currently unavailable
